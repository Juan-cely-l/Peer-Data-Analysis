## Abstract

This repository documents a peer data analysis exercise in which two unknown synthetic CSV files, `data15_big.csv` and `data15_small.csv`, are treated as "Alien Data": datasets whose generating logic is hidden from the analyst. The notebook is designed not only to describe the files, but to build a reproducible analytical framework for loading, profiling, cleaning, exploring, comparing, and statistically reverse-engineering their possible generative rules.

Although the task begins as exploratory data analysis, it quickly becomes a methodological exercise in inference under uncertainty. The central question is whether `data15_small.csv` is a reduced sample of `data15_big.csv`, a filtered variant, or a new realization generated by essentially the same stochastic process.

> Methodological position: empirical observation is kept separate from generative inference. Cleaning is intentionally conservative, and distributional claims are framed as heuristic evidence rather than ground truth.

## Research Scope

The notebook studies a synthetic dataset that resembles a vehicle registry from Colombia, with attributes related to identification, geography, technical configuration, capacity, insurance, inspections, and market value. The analytical design is driven by seven objectives:

1. Robustly load and validate both CSV files.
2. Profile their structure, data types, missingness, duplication, and cardinality.
3. Clean the data without destroying evidence of the underlying generator.
4. Perform univariate, bivariate, and multivariate exploratory analysis.
5. Infer plausible distribution families and signals of clipping, rounding, or mixture generation.
6. Detect functional dependencies, derived columns, and rule-based structures.
7. Compare `big` and `small` to infer whether they share the same generating mechanism.

## Analytical Methodology

### 1. Robust ingestion and validation

The notebook begins with a defensive loading strategy that:

- checks whether each file exists;
- detects delimiters automatically;
- tries alternative encodings when needed;
- estimates file size and expected memory pressure;
- decides between full loading and chunk-based sampling;
- records metadata about the effective loading decision.

This design is academically relevant because it makes computational constraints part of the documented methodology rather than an invisible implementation detail.

### 2. Structural profiling

Each dataset is profiled as a technical object before any transformation is made. The notebook records:

- dimensions and column names;
- inferred types at load time;
- null proportions;
- exact duplicates in the analysed frame;
- unique counts and high-cardinality fields;
- initial variable-level observations.

This stage establishes a reliable descriptive baseline and prevents premature interpretation.

### 3. Conservative cleaning

Cleaning is deliberately restrained. The notebook standardizes types, normalizes missing tokens, and builds cleaned working copies, but it does **not** automatically remove outliers or duplicates unless such a decision is methodologically justified. For synthetic data, anomalies may encode the generating logic itself; removing them too early would bias the inferential stage.

### 4. Exploratory data analysis

The exploratory component is organized at three levels:

- **Univariate analysis** for marginal distributions, dispersion, skewness, kurtosis, and dominant categories.
- **Bivariate analysis** for numerical correlations, contingency structures, and group-based comparisons.
- **Multivariate inspection** for near-deterministic relations, derived variables, and structural regularities.

The notebook combines descriptive tables, diagnostic plots, Pearson correlations, Cramer's V, and eta-squared to move from raw description to interpretable structure.

### 5. Reverse engineering logic

The core inferential layer attempts to answer questions such as:

- Is a variable continuous or discrete?
- Does it appear rounded, clipped, or truncated?
- Is there evidence of mixture behavior?
- Which distribution family is most plausible under heuristic model selection?
- Are some variables generated from others through deterministic or quasi-deterministic rules?

Goodness-of-fit comparisons rely on AIC, BIC, KS statistics, chi-squared diagnostics when appropriate, and visual evidence. The notebook explicitly warns that these results are heuristic because the same data are used both to estimate parameters and to evaluate fit.

### 6. Cross-dataset comparison

The final methodological layer compares the two files through:

- schema identity and type consistency;
- numerical distribution similarity;
- categorical distribution similarity;
- correlation-structure similarity;
- an automated but explicitly cautious hypothesis about the relationship between `big` and `small`.

## Dataset Profile

Across the cleaned analytical frames, the notebook identifies the same structural composition in both datasets:

- 38 shared columns with matching load-time types.
- 14 numerical variables.
- 13 categorical or boolean variables.
- 2 temporal variables.
- 9 identifier-like variables.
- No missing values in the analysed frames.
- No exact duplicates in the analysed frames.

The presence of many identifier-like fields such as `vehicle_id`, `vin`, `chassis_number`, `engine_number`, `soat_policy`, and `license_plate` is itself informative: these columns behave like generated keys rather than substantive analytical categories.

## Main Findings

### Distributional evidence

The notebook finds several stable marginal patterns across the two files:

- `engine_displacement` behaves like a positive continuous variable with approximate `0.1` rounding and is heuristically closest to a normal distribution.
- Many count-like variables such as `airbags`, `doors`, `horsepower_hp`, `seatbelts`, `seats`, `standing_capacity`, `wheels`, and `year` are classified by the automated fitting routine as discrete variables with Poisson-like signatures, often with evidence of clipping or mixture structure.
- `fuel_type` is dominated by Diesel at roughly `50%`, while gasoline remains the second largest category.
- `status` is highly concentrated in `Usado` at about `86.6%`, indicating a strongly imbalanced categorical rule.
- `vehicle_type` is nearly uniform across seven classes, suggesting deliberate balancing in the generator.
- `techinsp_result` is almost perfectly split between `Aprobado` and `Reprobado`, another sign of controlled synthetic construction.
- `emission_standard` is led by `Sin norma` at about `38.4%`, while the remaining regulatory classes are distributed much more evenly.

Taken together, these patterns strongly suggest engineered synthetic data rather than unconstrained natural observational data.

### Relational structure

The bivariate and multivariate stages reveal clear internal logic:

| Pattern | Evidence | Interpretation |
| --- | --- | --- |
| `seatbelts`, `seated_capacity`, and `seats` | Pairwise correlation = `1.000`, slope = `1.0`, intercept = `0.0` | These columns are effectively duplicated or directly derived from the same latent quantity. |
| `model -> brand` | Deterministic share = `1.0` | Model appears to be generated within brand-specific catalogs. |
| `municipality -> department` | Deterministic share = `1.0` | Geographic hierarchy is rule-based rather than noisy. |
| `engine_displacement` vs `horsepower_hp` | Correlation approx. `0.946` | Power is likely generated from displacement with limited noise. |
| `mileage_km` vs `year` | Correlation approx. `-0.848` | Newer vehicles systematically accumulate less mileage. |
| `price_cop` vs `year` | Correlation approx. `0.743` | Price increases strongly with recency. |
| `price_cop` vs `mileage_km` | Correlation approx. `-0.640` | Depreciation is structurally linked to accumulated use. |
| `doors` vs `vehicle_type` | Eta-squared = `1.000` | Vehicle type almost fully determines door count. |
| `emission_standard` vs `esp` | Cramer's V approx. `0.925` | Regulatory class and safety equipment are strongly linked. |
| `emission_standard` vs `abs` | Cramer's V approx. `0.919` | Brake-system logic is also embedded in the regulatory profile. |

This is one of the strongest conclusions of the notebook: the dataset is not merely random synthetic tabular data. It contains hierarchical assignments, deterministic mappings, and tightly constrained numerical relationships.

### Comparison between `big` and `small`

The comparative stage provides strong evidence that both files come from the same underlying generative design:

- Both datasets contain exactly the same 38 columns.
- All shared columns have the same inferred types at load time.
- `100.0%` of shared numerical variables are classified as very similar or similar with only minor changes.
- `68.2%` of shared categorical variables are classified as very similar or similar with only minor changes.
- The mean absolute difference between correlation matrices is only `0.0011`.

The categorical variables that differ the most are mostly identifier-like fields such as `vehicle_id`, `vin`, `chassis_number`, `engine_number`, `soat_policy`, `license_plate`, and `fur_code`. This is exactly what would be expected if unique keys were regenerated while the structural logic remained unchanged.

> Final comparative inference: `data15_small.csv` is most consistent with either a reduced sample from `data15_big.csv` or a new realization produced by an almost identical generative process. The notebook correctly leaves open the distinction between direct subsampling and regeneration with the same parameterization.

## Academic Conclusions

This project supports four substantive conclusions.

1. The data generation process is highly structured. Several columns are deterministic, near-deterministic, or constrained by explicit hierarchical rules.
2. The synthetic generator appears to combine balanced categorical design, engineered identifier fields, and rule-based numerical dependencies rather than simple independent random draws.
3. The two files are analytically coherent with one another and should be interpreted as products of the same underlying design family.
4. From a methodological perspective, the notebook demonstrates that conservative cleaning and layered exploratory analysis are effective for reverse-engineering hidden synthetic data generators.

In that sense, the notebook is not only a descriptive report but also a reproducible template for academic comparison between inferred logic and the true logic once it is revealed by the peer author.

## Limitations

The notebook is explicit about its limits:

- `data15_big.csv` was analysed through a reproducible sample because the raw file exceeds the interactive full-load threshold.
- Distribution-fitting diagnostics are heuristic and should be interpreted together with plots, not as definitive tests.
- Detection of derived variables and rule-based logic depends on automated thresholds and selected candidate relationships.
- The ultimate validation of all hypotheses requires comparison against the true generating process.
